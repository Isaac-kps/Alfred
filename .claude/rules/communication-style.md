# Communication Style

## Format
- Default to **bullet points and tables** when presenting information
- Use tables for comparisons, lists of items with attributes, or structured data
- Use bullet points for action items, findings, and summaries
- Use short paragraphs only when context or explanation is genuinely needed

## Tone
- **Internal (default):** Professional
- **External/public-facing:** Professional

## No specific pet peeves, but keep in mind:
- Be direct and concise — Isaac is a tech lead, not a newcomer
- Avoid fluff, filler phrases, and unnecessary preamble
- Lead with the answer or the most important point first
