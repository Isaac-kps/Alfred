# Skill: Morning Call

Pulls the current TradeTrust sprint from Jira and delivers a structured summary to Master Isaac.

## When to invoke

- When Master Isaac asks for a morning call or morning briefing
- At the start of a working session to get oriented

## What to do

1. Query the TradeTrust Jira board (project key: `TT`, cloud ID: `fbf183d8-5610-4a2a-9dae-5d3486e6be88`) for all issues in the current open sprint using JQL:
   ```
   project = TT AND sprint in openSprints() ORDER BY status ASC
   ```
2. Fetch fields: `summary`, `status`, `issuetype`, `priority`, `assignee`, `customfield_10028` (story points)
3. Produce a structured sprint summary in the following markdown format and save it to `report/sprint-summary/sprint-summary-<YYYY-MM-DD>.md` (use today's date). Create the directory if it does not exist.
4. Output the same summary to Master Isaac in the conversation.

### Report format

```markdown
# Sprint Summary — <YYYY-MM-DD>

**Project:** TradeTrust
**Generated by:** Alfred

---

## Sprint Overview
- Total issues: N
- In Development: N
- Dev Review: N
- QA Review / QA Done: N

- Canceled: N

### Story Points

| Done | N | N |

## Blockers / Highest Priority
| Key | Assignee | Summary |

...

## Master Isaac's Items

...

...

## Dev Review / QA

```

Keep it sharp and concise.
