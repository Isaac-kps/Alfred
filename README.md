# Alfred

Alfred is Isaac Koh's personal executive assistant and second brain — built on Claude Code and designed to operate with the precision and discretion of a distinguished butler.

---

## Who is Alfred?

Alfred is not a generic AI assistant. He is purpose-built for Master Isaac's role as Tech Lead at DEX — reducing cognitive overhead, increasing leverage, and serving as a reliable second brain across code quality, engineering leadership, sprint planning, and research.

> *"A good butler does not merely respond — he anticipates."*

---

## What Alfred Does

| Capability | Description |
|---|---|
| **Code Quality** | Scans DEX repositories, identifies quality issues, and produces structured improvement reports |
| **Research** | Conducts deep web research on any topic and saves structured reports to `research/` |
| **Morning Briefing** | Runs a weekday 9:00am briefing (Singapore time) — priorities, context, and what needs attention |
| **Decision Support** | Surfaces the right information at the right time; logs all meaningful decisions |
| **Sprint Planning** | Assists with issue tracking and sprint planning across TrustVC, TradeTrust, and OpenCerts |
| **Workflow Automation** | Turns recurring tasks into reusable skills stored in `.claude/skills/` |

## Project Structure

```
Alfred/
├── .claude/
│   ├── rules/              # Alfred's persona and behaviour rules
│   └── skills/             # Reusable skills (morning-call, research, ...)
├── context/                # Who Isaac is, what DEX does, current priorities, goals
├── decisions/              # Append-only decision log
├── documents/              # Reference documents (excluded from git)
├── frontend/               # React/Vite research browser (port 3003)
├── projects/               # Active workstreams (code-quality-poc, ...)
├── references/             # SOPs and example outputs
├── research/               # Research reports generated by Alfred
├── templates/              # Reusable templates (session summaries, etc.)
└── workspace/              # Cloned repositories for analysis (excluded from git)
```

---

## Skills

Skills are stored in `.claude/skills/` and can be invoked in any Claude Code session.

| Skill | Description |
|---|---|
| `/morning-call` | Schedules the weekday 9:00am morning briefing cron |
| `/research <topic>` | Researches a topic and saves a structured MD report to `research/` |

---

## Research Browser

Alfred ships with a local research viewer built in React + Vite.

```bash
cd frontend
npm install
npm run dev
# Opens at http://localhost:3003
```

Browse all research reports in the sidebar. Reports are served live from the `research/` directory.

## Context Files

Alfred's behaviour is shaped by the context files in `context/`:

- `context/me.md` — Who Isaac is
- `context/work.md` — DEX, projects, tools
- `context/team.md` — Team structure
- `context/current-priorities.md` — What's in focus right now
- `context/goals.md` — Q1 2026 goals and milestones

---

## Memory

Alfred maintains persistent memory across sessions. Preferences, feedback, and project context are stored in `.claude/projects/memory/` and automatically recalled in future conversations.
